Σε Windows

Οι παρακάτω οδηγίες έχουν δοκιμαστεί σε Windows 7 αλλά δεν υπάρχει λόγος να μη δουλεύουν και σε άλλες εκδόσεις.

Εγκατάσταση WinSCP

Κατέβασε το αρχείο εγκατάστασης

Μπες στη σελίδα http://winscp.net/eng/download.php#download2 και κατέβασε το αρχείο Installation Package με τη νεότερη έκδοση.
Κατά τη συγγραφή αυτού του οδηγού το νεότερο ήταν το winscp429setup.exe.

Εγκατάσταση του WinSCP

Τρέξε το αρχείο εγκατάστασης και ακολούθησε τα παρακάτω βήματα:

  1. Επίλεξε την γλώσσα της εγκατάστασης.
  2. Πάτησε "Next", και ξανά "Next".
  3. Επίλεξε "Typical installation" και "Next".
  4. Επίλεξε "Commander interface", "Next" και "Install".
  5. Κάνε επανεκκίνηση του υπολογιστή.

Σύνδεση

Τρέξε το WinSCP από το μενού Έναρξη, και επίλεξε την καρτέλα "Session" στα αριστερά. Συμπλήρωσε τα ακόλουθα στοιχεία:

Host name:
web-seminar.softlab.ntua.gr
Port number:
22
User name:
Το προσωπικό σου username
Password:
Το προσωπικό σου password

Το προσωπικό σου username και το προσωπικό σου password είναι τα ίδια που χρησιμοποίησες για την παράδοση των προηγούμενων εργασιών. Στη συνέχεια πάτα το κουμπί "Login".

Ενδέχεται επίσης την πρώτη φορά να δεις κάποια προειδοποίηση για την ταυτότητα του εξυπηρετητή. Αυτό συμβαίνει για λόγους ασφαλείας, ώστε να επιβεβαιώσεις ότι πράγματι συνδέθηκες εκεί που πιστεύεις ότι συνδέθηκες.

Ανέβασμα αρχείων

Αμέσως μετά τη σύνδεση, ο προεπιλεγμένος απομακρυσμένος φάκελος θα είναι ο προσωπικός σου φάκελος της μορφής /home/students/username όπου username είναι το προσωπικό σου όνομα χρήστη.

Στα δεξιά υπάρχουν τα αρχεία του απομακρυσμένου υπολογιστή. Στα αριστερά υπάρχουν τα αρχεία του τοπικού υπολογιστή. Για να ανεβάσεις κάποιο αρχείο, σέρνεις το επιθυμητό αρχείο από τα αριστερά προς τα δεξιά όπως στο Filezilla.

Αφού ανεβάσεις τα αρχεία σου, μπορείς αργότερα να τα αλλάξεις ξανά, να τα διαγράψεις, ή να τα αντικαταστήσεις.

Για να επιβεβαιώσεις ότι τα αρχεία σου έχουν ανέβει με επιτυχία, κλείσε το WinSCP, τρέξτο ξανά και συνδέσου και πάλι, και έλεγξε αν τα αρχεία που βλέπεις στον απομακρυσμένο υπολογιστή είναι τα σωστά.

Σε Ubuntu

Ανέβασμα αρχείων

Το Linux έχει συνήθως προεγκατεστημένα τα προγράμματα sftp και scp. Για να ανεβάσεις αρχεία, χρησιμοποίησε την εντολή scp από το Τερματικό του Linux. Για να τρέξεις το τερματικό του Linux, αν είσαι σε γραφικό περιβάλλον, μπορείς να πατήσεις Alt + F2 και να πληκτρολογήσεις xterm.

Η εντολή scp έχει την εξής μορφή:

scp τοπικό_αρχείο όνομα_χρήστη@απομακρυσμένος_υπολογιστής:απομακρυσμένη_τοποθεσία

Στην συγκεκριμένη περίπτωση, το τοπικό_αρχείο είναι το όνομα του αρχείου που θέλουμε να ανεβάσουμε. Βεβαιώσου ότι το αρχείο αυτό βρίσκεται στον τρέχοντα φάκελο ή ότι πληκτρολογείς την πλήρη διαδρομή αρχείου. Το όνομα_χρήστη είναι το όνομα χρήστη που έχεις χρησιμοποιήσει και στις προηγούμενες εργασίες (το πρώτο τμήμα του e-mail σου). Ο απομακρυσμένος_υπολογιστής είναι στην συγκεκριμένη περίπτωση το web-seminar.softlab.ntua.gr, ενώ η απομακρυσμένη_τοποθεσία είναι το ~ που αντιπροσωπεύει τον προσωπικό σου φάκελο.

Για παράδειγμα: scp test.html dionyziz@web-seminar.softlab.ntua.gr:~

Χρησιμοποιώντας χαρακτήρες-μπαλαντέρ, μπορεί κανείς να αντιγράψει πολλά αρχεία με μία εντολή. Για παράδειγμα με τον χαρακτήρα-μπαλαντέρ * στην θέση του τοπικό_αρχείο, αντιγράφονται όλα τα αρχεία του τρέχοντος φακέλου.

Πριν ανέβει το αρχείο, θα σου ζητηθεί να πληκτρολογήσεις τον προσωπικό σου κωδικό. Ενδέχεται επίσης την πρώτη φορά να δεις κάποια προειδοποίηση για την ταυτότητα του εξυπηρετητή. Αυτό συμβαίνει για λόγους ασφαλείας, ώστε να επιβεβαιώσεις ότι πράγματι συνδέθηκες εκεί που πιστεύεις ότι συνδέθηκες.

Έλεγχος

Για να ελέγξεις αν τα αρχεία σου έχουν ανεβεί σωστά, μπορείς να χρησιμοποιήσεις το πρόγραμμα sftp από το Τερματικό του Linux.

Η εντολή sftp έχει την εξής μορφή:

sftp όνομα_χρήστη@απομακρυσμένος_υπολογιστής

Αφού τρέξεις την εντολή, θα σου ζητηθεί να πληκτρολογήσεις τον προσωπικό σου κωδικό. Στη συνέχεια, μπορείς να χρησιμοποιήσεις την εντολή ls για να δεις την λίστα με τα αρχεία που έχεις ανεβάσει. Γράφοντας bye αποσυνδέεσαι.

Ασφάλεια και διορθώσεις

Αν είσαι έμπειρος χρήστης του Linux, σε ενθαρρύνουμε να αναζητήσεις προβλήματα ασφαλείας στο σύστημα παράδοσης εργασιών. Επειδή το SSH πρωτόκολλο επιτρέπει την απομακρυσμένη πρόσβαση σε κάποιον υπολογιστή, είναι πολύ πιθανό να έχουμε παραλήψει κάποια ρύθμιση ασφαλείας.

Σε περίπτωση που βρεις κάποιο πρόβλημα ασφαλείας, θα χαρούμε να μας το αναφέρεις για να το διορθώσουμε, και θα είναι τιμή μας να αναφέρουμε το όνομά σου στο μάθημα.

Σε περίπτωση που βρεις κάτι ελλειπές σε αυτές τις οδηγίες, τα σχόλιά σου είναι επίσης ευπρόσδεκτα μέσω e-mail.
